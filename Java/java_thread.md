# Thread (쓰레드)
- 프로그램안에서 단일한 연속적인 제어의 흐름
- 프로그램 안에 1개 혹은 그 이상의 Thread가 존재 할 수 있음
- 동시에 여러 개의 작업을 처리해야 할 필요성이 있을 경우 Muti Thread를 사용함
- process가 할당받은 자원을 이용하는 실행의 단위

## Process란?
- 운영체제로부터 자원을 할당받는 작업의 단위
	- Thread는 Process가 할당받은 자원을 Thread끼리 공유하며 실행 됨 
- 할당받는 자원
	- 주소공간, 메모리 등

##  Thread 사용의 장단점
### 장점
- 시스템의 throughput이 향상됨
- 시스템의 자원 소모가 줄어듬
- 프로세스 간 통신 방법에 비해 스레드 간의 통신 방법이 훨씬 간단함
### 단점
- 프로그램 디버깅이 어려움
- 단일 프로세서 시스템에서는 효과를 기대하기 어려움
- 여러 개의 스레드를 이용한 프로그램을 작성하는 경우 주의 깊게 설계해야 함
	- 미묘한 시간 차나 잘못된 변수를 공유함으로써 오류가 발생할 수 있음
